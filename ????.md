# Tianchi 二手车价格预测比赛总结

## 1. 这份文档的目的

这份文档用于快速说明三件事：

1. 这个比赛在做什么，评价标准是什么。
2. 当前仓库的训练流程如何组织，每个模块分别做什么。
3. 最终高分版本做了哪些关键改动，为什么这些改动有效。

本文以当前服务器最终保留版本为准：

- 最终稳定代码提交：`53c36fa`
- 最终文档收口提交：`68d73b2`
- 当前最佳本地 OOF：`Blend OOF MAE = 480.87277472593655`

## 2. 比赛任务

比赛目标是根据二手车的基础属性、匿名特征和时间信息，预测车辆成交价格 `price`。

核心评价指标是：

- `MAE`，即 Mean Absolute Error
- 分数越低越好

这类题目的重点通常不在超大模型，而在：

- 稳定的数据清洗
- 高质量特征工程
- 树模型参数调优
- 基于 OOF 的可靠融合

## 3. 当前项目的核心结构

当前仓库可以按下面理解：

- `feature/preprocess.py`
  负责类型转换、日期处理、数值裁剪、基础衍生特征、频次特征、目标统计特征、缺失值填充。

- `model/train.py`
  负责 LightGBM 和 CatBoost 的训练逻辑。

- `code/main.py`
  负责读取数据、准备训练分支、LightGBM 缓存、CatBoost 候选实验、OOF 融合、实验记录、输出预测文件。

- `user_data/`
  负责缓存和实验产物，例如：
  `metrics.json`
  `lgb_cache_predictions.npz`
  `lgb_cache_meta.json`

- `prediction_result/predictions.csv`
  最终提交文件，格式必须为 `SaleID,price`。

## 4. 整体训练流程

### 4.1 数据读取

训练入口在 `code/main.py`。

默认读取：

- `data/used_car_train_20200313.csv`
- `data/used_car_testA_20200313.csv`

之后统一进入 `build_features()`。

### 4.2 基础清洗与通用特征

`feature/preprocess.py` 先把原始表转换成适合树模型使用的稳定特征表。

主要处理包括：

- 数值列强制转换
- `regDate` / `creatDate` 日期解析
- 非法日期修补与非法标记
- 车龄相关特征：
  `car_age_days`
  `car_age_years`
  `date_gap_abs`
  `date_gap_negative`
- 发动机功率相关特征：
  `power_is_zero`
  `power_log1p`
  `power_per_km`
  `power_x_kilometer`
  `power_per_age`
- 里程对数特征：
  `kilometer_log1p`
- 匿名 `v_0 ~ v_14` 的聚合特征：
  `v_sum`
  `v_mean`
  `v_std`
  `v_min`
  `v_max`
  `v_range`
- 组合类别特征：
  `brand_model`
  `brand_bodyType`
- 缺失数量特征：
  `missing_count`

这一阶段的核心思想是：

- 对长尾数值做裁剪和对数变换
- 把原始日期改造成可解释时间差
- 用少量交互特征帮助树模型切分价格区间

### 4.3 频次特征

之后通过 `add_frequency_features()` 生成类别频次特征，例如：

- `brand_freq`
- `model_freq`
- `name_freq`
- `brand_model_freq`

这类特征让模型知道：

- 某个类别是否常见
- 类别出现频率和价格稳定性之间的关系

### 4.4 两条训练分支

当前方案不是一个单模型单特征表，而是两条互补分支。

#### LightGBM 分支

LightGBM 分支在基础特征和频次特征之上，再加入折内目标统计特征。

关键特征组包括：

- `brand_price_mean`
- `model_price_median`
- `brand_model_price_count`
- 以及 `brand`、`model`、`bodyType`、`fuelType`、`gearbox`、`regionCode`、组合类别的多种目标统计量

这些特征通过 `add_fold_target_stat_features()` 构造，严格使用折内训练集统计，避免目标泄漏。

#### CatBoost 分支

CatBoost 分支保持相对干净的增强特征表，不叠加折内 target stats。

这样做的原因是：

- CatBoost 本身就擅长处理类别特征
- 与 LightGBM 保持分支差异，有利于后续融合

### 4.5 缺失值处理

最后通过 `fill_missing_values()` 处理缺失值：

- 类别列填 `"missing"`
- 数值列填训练集该列中位数

### 4.6 五折 OOF 训练

所有实验统一采用：

- `5` 折交叉验证
- 固定随机种子

这样做的价值是：

- 所有实验都以 OOF MAE 为统一比较标准
- 每轮实验结果具有严格可比性

### 4.7 LightGBM 训练逻辑

LightGBM 使用的是对数目标回归：

- 训练目标：`log1p(price)`
- 预测后：`expm1` 还原

最终保留参数为：

- `n_estimators=4000`
- `learning_rate=0.03`
- `num_leaves=127`
- `subsample=0.8`
- `colsample_bytree=0.8`
- `reg_alpha=0.2`
- `reg_lambda=0.2`

### 4.8 CatBoost 训练逻辑

CatBoost 最终采用直接回归原始价格，而不是对数目标。

最终保留参数为：

- `iterations=4600`
- `learning_rate=0.03`
- `depth=9`
- `l2_leaf_reg=9.0`
- `od_wait=140`

对应候选名：

- `cfg_w_depth_9_iter4600`

### 4.9 OOF 融合

融合不是手动写死，而是用 OOF 预测搜索最优权重。

最终保留版本中：

- LightGBM 权重：`0.282`
- CatBoost 权重：`0.718`

搜索分辨率已经从早期的 `0.01` 细化到了 `0.001`。

### 4.10 LightGBM 缓存机制

为了加速连续实验，项目增加了 LightGBM 缓存：

- `prepare_lgb_cache`
- `user_data/lgb_cache_predictions.npz`
- `user_data/lgb_cache_meta.json`

这让只改 CatBoost 参数时，不必重复训练 LightGBM。

## 5. 各阶段实验思路复盘

### 阶段 1：先搭稳定基线

最先解决的是工程问题，而不是暴力提分：

- 统一训练入口
- 固定交叉验证
- 固定指标来源
- 固定预测输出格式

没有稳定基线，就没有可靠实验。

### 阶段 2：构建稳定特征空间

这一阶段主要完成：

- 日期特征
- 车龄特征
- 功率和里程交互
- 组合类别特征
- 缺失计数特征
- 类别频次特征

核心思路是：

- 用稳定、可解释、训练测试一致的特征
- 不做过度复杂的堆特征

### 阶段 3：加入折内目标统计特征

这是整套方案的重要提分点之一。

原因在于：

- 二手车价格和品牌、车型、地区、车身结构高度相关
- 目标统计特征能有效把高基数类别映射到更有预测力的数值空间

但必须折内构造，否则会发生泄漏。

### 阶段 4：双分支差异化

LightGBM 和 CatBoost 不使用完全相同的表征方式。

这一步的意义在于：

- LightGBM 更偏向 target stats 和数值增强
- CatBoost 更偏向原始类别能力
- 两者差异化后，融合收益会更稳定

### 阶段 5：引入缓存，提升实验效率

这一步本身不是直接提分，而是让实验系统更高效。

有了缓存后，可以高频率做：

- CatBoost 单候选实验
- 小范围 LightGBM follow-up

这对后期连续提分非常关键。

### 阶段 6：围绕 CatBoost 强分支做单变量微调

这一阶段遵循严格策略：

- 一次只改一个参数
- 只与当前稳定基线比较
- 好就保留，差就回退

这一阶段尝试过：

- `l2_leaf_reg` 微调
- `od_wait` 微调
- `iterations` 增加
- `depth` 调整

最终有效方向包括：

- `depth 8 -> 9`
- `iterations 4400 -> 4600`

### 阶段 7：轻量特征扩展尝试

尝试过的方向之一是：

- 车龄归一化 usage 特征

结果是：

- LightGBM 和最终 blend 都变差

这说明当前特征空间已经较成熟，再增加这类组合项反而引入噪声。

### 阶段 8：LightGBM 跟进调优

在 CatBoost 分支强化后，回到 LightGBM 做单变量跟进。

最有效的一步是：

- `num_leaves 95 -> 127`

但后续正则增强方向失败：

- `reg_lambda 0.2 -> 0.3` 失败
- `reg_alpha 0.2 -> 0.3` 失败

这说明：

- LightGBM 有结构容量空间
- 但在当前特征空间下，并不需要更重的正则

### 阶段 9：融合精度细化

将融合权重搜索步长从 `0.01` 提升到 `0.001` 后，得到了一次小幅但真实的提分。

这个结果说明：

- 前面的主模型已经很强
- 融合层面仍有一点点可榨取的边际收益

### 阶段 10：最终突破

在 LightGBM 正则方向连续失败、blend 细化只带来小幅提升之后，重新回到 CatBoost 最强分支。

最终成功动作是：

- 在 `depth=9` 的稳定分支上，把 `iterations` 从 `4400` 提到 `4600`

这一步将分数最终推进到：

- `Blend OOF MAE = 480.87277472593655`

## 6. 最终高分版本的关键措施

最终高分版本并不是依赖某一个单独技巧，而是多次正确修改叠加后的结果。

### 6.1 最终保留配置

#### LightGBM 分支

- 使用 `log1p(price)` 训练
- 使用折内目标统计特征
- 使用类别频次特征
- `num_leaves=127`
- `reg_alpha=0.2`
- `reg_lambda=0.2`

#### CatBoost 分支

- 直接预测原始价格
- `depth=9`
- `iterations=4600`
- `l2_leaf_reg=9.0`
- `od_wait=140`

#### 融合

- 用 OOF 搜索最优权重
- 搜索步长细化到 `0.001`
- 最优权重：
  LightGBM `0.282`
  CatBoost `0.718`

### 6.2 为什么这些改动有效

可以概括为四点：

#### 1. 两条模型分支有互补性

LightGBM 和 CatBoost 没有完全同质化：

- 一个更吃统计特征
- 一个更吃类别建模和结构调优

这让融合真正有收益。

#### 2. LightGBM 的特征空间和容量匹配更好

有了 target stats 之后，`num_leaves=127` 让模型更有能力利用这些特征。

#### 3. CatBoost 深度 9 分支是主增益来源

最终从 `cfg_t_depth_9_iter4400` 到 `cfg_w_depth_9_iter4600` 的推进，是本阶段跨过目标阈值的决定性改动。

#### 4. 融合精度细化榨干了边际收益

`0.001` 级别权重搜索贡献不大，但在高分阶段很有价值。

## 7. 最关键的实验结果

### 成功保留的关键实验

1. `task4_lgb_leaves127_cfg_t_eval`
   `num_leaves 95 -> 127`
   `Blend OOF MAE = 481.39377557629706`

2. `task7_blend_fine1000_cfg_t_eval`
   blend 搜索精度 `0.01 -> 0.001`
   `Blend OOF MAE = 481.39230680178133`

3. `task8_cat_iter4600_cfg_w_eval`
   CatBoost `iterations 4400 -> 4600`
   `Blend OOF MAE = 480.87277472593655`

### 明确失败、后续可避免的方向

1. `task3_age_usage_cfg_t_eval`
   age-normalized usage features

2. `task5_lgb_reglambda03_cfg_t_eval`
   `reg_lambda 0.2 -> 0.3`

3. `task6_lgb_regalpha03_cfg_t_eval`
   `reg_alpha 0.2 -> 0.3`

4. `task1_l2_10_depth9_eval`
   CatBoost `l2_leaf_reg 9 -> 10`

5. `task2_od160_depth9_eval`
   CatBoost `od_wait 140 -> 160`

## 8. 如何快速读懂当前代码

建议按下面顺序理解：

1. 先读 `feature/preprocess.py`
   看原始字段是怎样被加工成模型输入的。

2. 再读 `model/train.py`
   看 LightGBM 和 CatBoost 的训练目标、参数和 CV 逻辑。

3. 最后读 `code/main.py`
   看训练编排、缓存、候选实验、融合搜索和产物输出。

不要反过来。

如果先看 `main.py`，容易只看到流程控制，却不知道这些控制服务于什么特征和什么建模决策。

## 9. 一句话总结

这套高分方案本质上是：

`稳定特征工程 + LightGBM / CatBoost 差异化双分支 + 折内 target stats + OOF 融合 + 单轮单改动实验纪律`

最终高分不是偶然，而是这套纪律化实验流程逐步筛选出来的结果。

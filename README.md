# 天池二手车交易价格预测项目

> 服务器端唯一维护版本的比赛工程，目标是在可回退、可复现、可持续迭代的前提下，把二手车价格预测的本地 OOF MAE 稳定压低。

## 项目概览

本项目对应天池二手车交易价格预测赛题，任务是基于车辆属性、匿名特征与交易上下文，预测测试集车辆的成交价格。当前仓库已经形成完整的服务器端实验闭环：特征构建、交叉验证、单模型训练、OOF 融合、结果记录、优胜保留、失败回退，全部在同一台服务器上完成。

这个版本不是“从零搭一个 baseline”，而是围绕稳定基线持续做小步提分。核心方法不是追求复杂模型堆叠，而是把表格特征工程、树模型调参与 OOF 融合做扎实。

## 方法总览

### 1. 问题建模

这是一个标准的表格回归任务，评价指标是 `MAE`。数据存在几个典型特征：

- 数值字段与类别字段混合存在。
- 日期字段同时包含“注册时间”和“上架时间”。
- 含有 15 个匿名特征 `v_0 ~ v_14`。
- 目标价格分布长尾明显，不同品牌、车型、车龄之间差异大。

因此，主线策略不是上深度模型，而是优先使用对表格数据更稳的 GBDT 系模型。

### 2. 特征工程

特征工程集中在 `feature/preprocess.py`，整体原则是“低泄漏、可解释、训练测试一致”。当前有效特征主要包括：

- 日期解析与车龄特征：`regDate`、`creatDate`、`car_age_days`、`car_age_years`、`date_gap_abs`、`date_gap_negative`
- 数值变换特征：`power_log1p`、`kilometer_log1p`、若干功率与里程、车龄之间的比值或交互
- 匿名变量聚合特征：`v_sum`、`v_mean`、`v_std`、`v_min`、`v_max`、`v_range`
- 组合类别特征：如 `brand_model`、`brand_bodyType`
- 频次特征：如 `brand_freq`、`model_freq`、`name_freq`、`brand_model_freq`
- 缺失信息特征：如 `missing_count`
- 面向 LightGBM 的折内目标统计特征

高分版本的关键点之一，是对不同模型使用不同信息密度的特征集合，而不是强行共用同一份输入。

### 3. 模型设计

#### LightGBM

LightGBM 分支承担两个职责：

- 作为快速、稳定的树模型基线。
- 作为融合中的“辅助视角”，补充 CatBoost 没有覆盖好的样本结构。

最终保留参数为：

```text
n_estimators=4000
learning_rate=0.03
num_leaves=127
subsample=0.8
colsample_bytree=0.8
reg_alpha=0.2
reg_lambda=0.2
```

这一分支使用 `log1p(price)` 训练，并结合折内目标统计特征，优先保证验证过程与真实推理过程一致。

#### CatBoost

CatBoost 是当前最强单模型，也是最终成绩的主要来源。它更擅长处理该任务中存在的类别信息、非线性关系以及噪声较多的表格结构。

最终保留候选为：

```text
cfg_w_depth_9_iter4600
iterations=4600
learning_rate=0.03
depth=9
l2_leaf_reg=9.0
od_wait=140
```

从实验记录看，CatBoost 在这个任务中对“是否使用 `log1p(price)`”“是否接入目标统计特征”“迭代轮数与正则强度的平衡”都比较敏感，因此后续提分基本都围绕它做单变量细化。

### 4. OOF 融合

最终结果不是直接取单模最优，而是基于 5 折 OOF 预测做权重搜索。这个流程有两个关键价值：

- 不依赖主观经验拍脑袋设权重。
- 可以直接用 OOF MAE 判断融合是否真的带来增益。

最终最优权重搜索细化到 `0.001` 步长，最佳组合为：

- LightGBM：`0.282`
- CatBoost：`0.718`

这一步本身带来的提升不算巨大，但属于低风险收益，且能稳定服务于后续模型更新。

## 提分路径摘要

整个项目的提分不是一次性跃迁，而是沿着下面这条主线逐步压低误差：

1. 先搭通完整工程与双模型 baseline。
2. 加入基础统计与 `log1p` 目标变换，初步改善长尾问题。
3. 把目标统计特征改成折内生成，消除验证泄漏风险。
4. 验证 LightGBM 与 CatBoost 是否应该分开使用不同特征集。
5. 确认 CatBoost 适合回到原始 `price` 目标空间。
6. 细化 LightGBM 结构参数，尤其是 `num_leaves`。
7. 细化融合权重搜索粒度。
8. 持续推进 CatBoost 单变量实验，最终把 `iterations` 提升到 `4600`、`depth` 提升到 `9`，得到当前最佳版本。

从结果上看，真正有效的提分主要来自三类动作：

- 让验证协议更严格而不是更乐观。
- 让 CatBoost 的目标空间和参数更贴合数据分布。
- 让融合成为“锦上添花”，而不是“掩盖单模问题”。

## 实验纪律

服务器端实验遵循固定规则：

- 所有修改、训练、提交、推送都只在服务器仓库执行。
- 每轮只改一个明确方向，避免多变量混杂。
- 结果变好才保留代码；结果变差则回退代码，但日志必须保留。
- 正式比较只看 `user_data/metrics.json` 中的 `blend_oof_mae`。
- 失败实验不删除记录，因为它们同样构成有效的搜索边界。

这个纪律本身就是项目的一部分。它保证当前高分不是偶然跑出来的，而是可解释、可复现、可回退的产物。

## 结论

当前版本的竞争力，不来自复杂堆料，而来自一套执行得比较干净的表格建模方法：

- 特征工程尽量稳定、低泄漏。
- LightGBM 与 CatBoost 各司其职。
- 融合建立在可靠 OOF 之上。
- 实验过程强调单变量、可回退、可记录。

如果后续继续优化，最值得优先尝试的方向仍然是：围绕 CatBoost 做更细的单变量参数实验，以及在不破坏验证纪律的前提下增加少量高质量轻量特征。

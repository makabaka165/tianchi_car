# 记录

## 第 1 轮：项目骨架 + baseline 跑通

### 目标

- 按赛题提交规范搭建可复现工程结构
- 解决原始数据读取错位问题
- 完成第一版从训练到 A 榜预测文件输出的全流程

### 新建/修改内容

- 新建 `feature/loader.py`
  - 使用 `csv.reader(delimiter=" ")` 保留空字段
  - 将 `""` 和 `"-"` 统一为缺失值
- 新建 `feature/preprocess.py`
  - 数值类型转换
  - `regDate` / `creatDate` 日期解析
  - 车龄、日期拆分、频次特征、组合特征
  - 删除低信息特征 `offerType`
- 新建 `model/train.py`
  - 5 折 `LightGBMRegressor`
  - 5 折 `CatBoostRegressor`
- 新建 `code/main.py`
  - 串联读取、特征、训练、融合、生成预测结果
- 新建 `code/requirements.txt`
- 新建 `README.md`

### 算法思路

- 问题类型：表格回归
- 模型路线：
  - LightGBM baseline
  - CatBoost baseline
  - 50/50 简单加权融合
- 验证方式：
  - 5 折 KFold
- 特征：
  - 原始数值与类别特征
  - 匿名特征 `v_0` ~ `v_14`
  - 日期拆分
  - 车龄特征
  - 频次特征
  - `brand_model` / `brand_bodyType` 组合特征

### 效果

- LightGBM OOF MAE: `556.9011`
- CatBoost OOF MAE: `509.5204`
- 生成文件：
  - `prediction_result/predictions.csv`
  - `user_data/metrics.json`

### 结论

- 数据读取问题已经稳定解决
- CatBoost 明显优于 LightGBM
- 下一轮优化优先围绕：
  - 目标变换
  - 统计特征
  - 更合理的验证与融合

## 第 2 轮：统计特征 + log1p 目标变换

### 目标

- 在不改工程结构的前提下提升 baseline
- 强化价格统计信息
- 观察 `log1p(price)` 对 MAE 的影响

### 新建/修改内容

- 修改 `feature/preprocess.py`
  - 新增 `add_target_stat_features`
  - 针对 `brand`、`model`、`bodyType`、`fuelType`、`gearbox`、`regionCode`
  - 以及 `brand_model`、`brand_bodyType`
  - 增加价格统计特征：
    - `mean`
    - `median`
    - `min`
    - `max`
    - `count`
- 修改 `model/train.py`
  - 为 LightGBM / CatBoost 增加 `use_log_target` 开关
  - 支持使用 `log1p(price)` 训练，再 `expm1` 还原预测
- 修改 `code/main.py`
  - 接入目标统计特征
  - 第二轮使用 `log1p(price)` 训练
  - 融合权重改为 `0.3 * LightGBM + 0.7 * CatBoost`

### 算法思路

- 目标统计特征：
  - 对高信息类别字段注入价格先验
  - 本质是把训练集中的局部价格分布压缩成可学习信号
- `log1p(price)`：
  - 价格分布长尾明显
  - 对数变换可以减轻极端高价样本的影响
- 融合策略：
  - 继续保留双模型
  - 但在权重上明显偏向更强的 CatBoost

### 效果

- LightGBM OOF MAE: `540.6442`
  - 相比第 1 轮 `556.9011`，有明显提升
- CatBoost OOF MAE: `510.3384`
  - 相比第 1 轮 `509.5204`，略有回退
- 特征数：
  - 从 `52` 增加到 `92`

### 结论

- `log1p + 统计特征` 对 LightGBM 很有效
- CatBoost 在当前参数下没有同步获益，说明它对新增统计特征和目标变换更敏感
- 下一轮更值得做的是：
  - 用 OOF 自动搜索最优融合权重
  - 单独为 CatBoost 调参
  - 将统计特征改成更严格的折内生成，进一步控制泄漏风险

## 第 3 轮：折内统计特征 + OOF 自动融合

### 目标

- 消除第 2 轮统计特征的验证泄漏风险
- 让特征生成的折与模型验证的折保持一致
- 不再手工指定融合权重，改为用 OOF 自动搜索

### 新建/修改内容

- 修改 `feature/preprocess.py`
  - 新增 `TARGET_STAT_GROUPS`
  - 新增 `TARGET_STAT_AGGS`
  - 新增 `add_fold_target_stat_features`
  - 对训练集使用折内价格统计特征
  - 对测试集使用全量训练集统计特征
- 修改 `model/train.py`
  - LightGBM 增加早停
  - CatBoost 增加早停
  - 支持直接传入外部固定折划分
  - 收敛 CatBoost 参数以控制训练时间
- 修改 `code/main.py`
  - 用固定 `KFold` 先生成折
  - 折划分同时用于统计特征生成和模型训练
  - 新增 `search_blend_weight`
  - 基于 OOF 在 `[0, 1]` 范围内网格搜索最优 CatBoost 融合权重

### 算法思路

- 第 2 轮的统计特征是直接由全训练集聚合生成的，验证会偏乐观
- 第 3 轮改成：
  - 每个验证折只能看到其对应训练折的价格统计
  - 这样得到的 OOF MAE 更接近真实泛化能力
- 融合部分：
  - 不预设 LightGBM / CatBoost 权重
  - 直接用 OOF 搜权重
  - 以整体 MAE 最小为准

### 效果

- LightGBM OOF MAE: `540.4576`
  - 相比第 2 轮 `540.6442`，小幅提升
- CatBoost OOF MAE: `543.4157`
  - 相比第 2 轮 `510.3384`，明显变差
  - 说明第 2 轮 CatBoost 结果受到了统计特征泄漏影响
- 自动搜索后的最优融合权重：
  - LightGBM: `0.5`
  - CatBoost: `0.5`
- 融合 OOF MAE: `517.7229`

### 结论

- 第 3 轮的验证结果更可信
- LightGBM 在折内统计特征下相对稳定
- CatBoost 当前配置不适合这组更严格的统计特征
- 虽然单模型没有明显变强，但融合后的 OOF MAE 优于两者单独结果，说明模型互补性仍然存在
- 下一轮更值得做的是：
  - 为 CatBoost 单独瘦身特征或改回非统计特征版本对比
  - 基于 OOF 做更细粒度的融合权重搜索
  - 尝试按时间切分做辅助验证，检查线上分布偏移
